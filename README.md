# cse305_parallel_computational_economics

Within this repository, you will find our project on parallel computational economics as part of the Concurrent and Distributed Computing (CSE305) course.  
Overall, we implement and then parallelize an agent-based financial market model inspired by Lux and Marchesi, *Scaling and criticality in a stochastic multi-agent model of a financial market*.

## Objective

Our goal is to simulate a stochastic multi-agent financial market and study how the computation can be parallelized on CPU.

Throughout the project, we propose:

- a sequential implementation of the model;
- a CPU-parallel implementation;
- benchmarks (runtime, speedup...) for different numbers of agents, time steps, and threads;

## Model

The model is inspired by Lux and Marchesi's stochastic multi-agent financial market model. The simulated market contains heterogeneous agents with different trading rules:

- **fundamentalists** trade according to the deviation between the market price and a fundamental value;
- **noise traders**, or chartists, react to price trends and market sentiment. They are divided into optimistic and pessimistic traders.

The model combines three mechanisms:

1. stochastic changes in the fundamental value;
2. endogenous switching of agents between trading states;
3. endogenous price changes caused by aggregate excess demand.



## Parallelization strategy

The sequential implementation is used as a reference.

The CPU-parallel version focuses on parallelizing operations over agents, especially:

- stochastic state updates;
- local computation of agent counts;
- local computation of excess demand;
- reduction of thread-local quantities into global market variables.

## Detailed model implementation

The implementation follows the main mechanisms described by Lux and Marchesi, while using explicit discrete-time rules suitable for simulation. The market contains three agent states:

- `Optimist`: noise trader expecting a rising market;
- `Pessimist`: noise trader expecting a declining market;
- `Fundamentalist`: trader expecting the market price to revert toward the fundamental value.

The main state variables are:

| Variable | Meaning |
|---|---|
| `p` | current market price |
| `pf` | current fundamental value |
| `r` | log-return of the market price |
| `epsilon` | exogenous shock to the fundamental value |
| `n_optimists` | number of optimistic noise traders |
| `n_pessimists` | number of pessimistic noise traders |
| `n_fundamentalists` | number of fundamentalists |
| `excess_demand` | aggregate excess demand generated by all agents |

At each time step, the simulation performs the following operations:

1. **Update the fundamental value.**  
   The fundamental value follows an exogenous stochastic process. Its log-increments are Gaussian:
   `log(pf_t) - log(pf_{t-1}) = epsilon_t`, where `epsilon_t` is normally distributed.

2. **Compute market indicators.**  
   The simulation computes the current log-return, the deviation between market price and fundamental value, and the majority opinion among noise traders.

3. **Update agent states.**  
   Transitions follow the Lux--Marchesi form

   \[
   \mathbb{P}(i \to j \text{ during } \Delta t)
   =
   \nu_{ij}\exp(U_{ij}(t))\Delta t,
   \]

   where \(\nu_{ij}\) is a revision frequency and \(U_{ij}(t)\) is the forcing term.

   For opinion changes among noise traders, let

   \[
   x_t = \frac{n_+(t)-n_-(t)}{n_+(t)+n_-(t)},
   \qquad
   r_t = \log\left(\frac{p_t}{p_{t-1}}\right),
   \]

   where \(x_t\) measures market sentiment and \(r_t\) is the current price trend. We use

   \[
   U_{-\to +}(t)=a_1x_t+a_2r_t,
   \qquad
   U_{+\to -}(t)=-a_1x_t-a_2r_t.
   \]

   For switches between noise traders and fundamentalists, transitions are driven by relative profitability. We use the proxies

   \[
   \pi_+(t)=r_t,\qquad
   \pi_-(t)=r_{\mathrm{alt}}-r_t,\qquad
   \pi_f(t)=\delta\left|\log\left(\frac{p_{f,t}}{p_t}\right)\right|,
   \]

   with \(0<\delta<1\). The forcing terms are profit differences:

   \[
   U_{f\to +}=\pi_+-\pi_f,\quad
   U_{+\to f}=\pi_f-\pi_+,
   \]

   \[
   U_{f\to -}=\pi_- - \pi_f,\quad
   U_{-\to f}=\pi_f-\pi_-.
   \]

   Agents are therefore more likely to switch toward the state with the higher current profitability proxy.


4. **Compute excess demand.**  
   Aggregate excess demand is the sum of noise-trader demand and fundamentalist demand:

   \[
   ED_t = ED_t^{n} + ED_t^{f}.
   \]

   Noise traders have a constant average trading volume \(q_n\), so their demand is determined by the difference between optimistic and pessimistic traders:

   \[
   ED_t^{n} = q_n\left(n_+(t)-n_-(t)\right).
   \]

   Fundamentalists trade against mispricing. We use a log-deviation from the fundamental value:

   \[
   ED_t^{f} = q_f n_f(t)\log\left(\frac{p_{f,t}}{p_t}\right).
   \]

   Hence, if \(p_t < p_{f,t}\), fundamentalists buy; if \(p_t > p_{f,t}\), they sell.



5. **Update market price.**  
   The market price changes endogenously as a response to excess demand. We use a log-price impact rule:

   \[
   p_{t+1}
   =
   p_t \exp\left(\beta \frac{ED_t}{N}\right),
   \]

   where \(\beta\) is the price-impact parameter and \(N\) is the total number of agents. Hence, positive excess demand increases the price, while negative excess demand decreases it. This multiplicative rule keeps prices strictly positive.

   This price-update rule is our discrete-time implementation of the excess-demand mechanism described in the paper.


6. **Record outputs.**  
   The simulation stores prices, fundamental values, returns, agent counts, and excess demand for later analysis and benchmarking.

The main parallelization challenge is that agent updates are naturally local, but market variables such as excess demand, agent counts, and price updates are global. The CPU-parallel implementation therefore has to split agents across threads, compute thread-local quantities, and then reduce them into global market variables at each time step.
